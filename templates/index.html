Short guide

  1. Create a new Droplet (Digital Ocean's term for a VPS) with the following specifications:
    • Under 'Choose an image':
      • Select Ubuntu 20.04 (LTS) x64. Ubuntu is currently the most popular Linux distribution.
    • Under 'Choose a plan':
      • Select Basic. This falls under the Shared CPU category, which means the machine you get access to is entirely virtual. It runs on a physical machine together with a number of other virtual machines that should never learn anything about each other.
      • Select $5/mo.
    • Under 'Choose datacenter region' choose a datacenter near you. This results in the least latency when you're working on the server. In practice you will pick this based on where your customers are located.
    • Don't select any additional options.
    • Under 'Authentication', select 'Password' and create a secure password.
  1. Name your project.
  1. Copy the IP address that belongs to your Droplet.
  1. In a terminal, run the following command, where you replace <ip> with the IP address you copied:
ssh root@<ip>
  1. If asked, accept the authenticity of the host by typing 'yes'.
  1. When asked for a password, fill in the password you configured for this Droplet and hit enter.
  1. You should now be logged in to the server successfully! If so, you're greeted by a message that starts with:
Welcome to Ubuntu 20.04.1 LTS (GNU/Linux 5.4.0-51-generic x86_64)
  1. apt update -- this updates the list of packages and their version that known to apt at the moment.
  1. apt upgrade -- this starts the process of updating all packages on the system to the latest version.
  1. Install nginx:
sudo apt install nginx
    sudo systemctl enable nginx
    sudo systemctl start nginx

  1. Create a new github repository to work in on your local machine called CD and paste this text into a file main.py:
# Import what we need from flaskfrom flask import Flask
    # Create a Flask app inside `app`app = Flask(__name__)
    # Assign a function to be called when the path `/` is requested@app.route('/')
    def index():
    return 'Hello, world!'@app.route('/cow')
    def cow():
    return 'MOoooOo!'

  1. Copy your folder to the VPS using scp. The command looks like this:
# -r for recursive, because we're copying a folder
    scp -r cd root@<SERVER_IP>:/home/

  1. Verify that your Flask app has arrived safely by logging into your server over SSH and checking farm and all its contents is where you expect it to be -- in /home/.
  1. Install Flask and Gunicorn on the VPS:
pip install gunicorn --upgrade
    pip install flask

  1. Create a new file:
 touch /etc/systemd/system/cd.service
  1. Open the file in the nano command line text editor:
nano /etc/systemd/system/cd.service
  1. Paste this in there:
[Unit]
    # This could be anything that helps you and colleagues know what this
    # service is for.
    Description=winc-cd gunicorn daemon
    # This tells systemd when this application is ready to start
    After=network.target
    
    [Service]
    Type=notify
    DynamicUser=yes
    RuntimeDirectory=winc-cd
    # Where the command supplied in ExecStart be run
    WorkingDirectory=/home/winc-cd/
    ExecStart=/usr/local/bin/gunicorn main:app
    ExecReload=/bin/kill -s HUP $MAINPID
    Restart=on-failure
    KillMode=mixed
    TimeoutStopSec=5
    PrivateTmp=true
    
    [Install]
    WantedBy=multi-user.target

  1. Run:
systemctl enable --now cd
    systemctl status cd

  1. Verify:
curl 127.0.0.1:8000
    # Output: Hello, world!

  1. Overwrite /etc/nginx/sites-available/default with this:
server {
        # Listen on port 80
        listen 80;
    
        server_name winc-cd;
    
        access_log  /var/log/nginx/access.log;
        error_log  /var/log/nginx/error.log;
    
        # Apply the following for requests to `/` and beyond
        location / {
            # Pass the request on to this local IP
            # This is where our Gunicorn server is listening
            proxy_pass         http://127.0.0.1:8000/;
            proxy_redirect     off;
    
            proxy_set_header   Host                 $host;
            proxy_set_header   X-Real-IP            $remote_addr;
            proxy_set_header   X-Forwarded-For      $proxy_add_x_forwarded_for;
            proxy_set_header   X-Forwarded-Proto    $scheme;
        }
    }

  1. Run:
systemctl restart nginx

  1. To verify go to the servers IP address in the browser.
  1. Write a test for the flask app in the same directory as the flask app.
  1. Make a requirements.txt file with modules that need to be installed in order for you flask app to work:
pytest==6.1.2
    flask

  1. Create a file called run_tests.yml and place it so the repository looks like this:
.
    ├── .github
    │   └── workflows
    │       └── run-tests.yml
    ├── requirements.txt
    ├── lib.py
    └── test_lib.py

  1. Place this content in run-tests.yml:
name: Run tests
    
    # Run this workflow whenever something new is pushed.
    on: push
    jobs:
      run-tests:
        runs-on: ubuntu-20.04
        steps:
          - name: Checkout repository
            uses: actions/checkout@v2
          - name: Setup Python
            uses: actions/setup-python@v2
            # Specify some input for this particular action
            with:
              python-version: '3.8.6'
          - name: Install Dependencies
            run: pip install -r requirements.txt
          - name: Run tests
            run: pytest

  1. Commit and push all your changes. It should immediately run.
  1. Go to the repository on Github and check under the actions tab to see if the test passed.
